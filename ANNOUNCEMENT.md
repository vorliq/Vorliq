Reddit Announcement
===================

Vorliq Version 1.0 is live. Vorliq is experimental open-source community blockchain software built on its own lightweight chain. The native coin is called VLQ, and the project includes a Python blockchain core, Node.js backend, React web app, React Native mobile app, peer networking, wallet tools, mining, community voting, exchange offers, governance, and public documentation.

The problem Vorliq explores is practical community coordination. Many cryptocurrency projects are expensive, complex, or dependent on large outside networks. Vorliq does not depend on Ethereum, Bitcoin, Solana, or any external blockchain. It lets people test wallets, signed transactions, public records, community requests, governance proposals, and node operation in one open-source system.

VLQ has a maximum supply of 21 million coins, a starting mining reward of 50 VLQ per block, and a scheduled halving every 210000 blocks. VLQ has no guaranteed market value, is not a regulated investment product, and should not be treated as a promise of profit or income. Mining rewards, lending-style activity, exchange offers, treasury activity, tips, and governance settings are experimental software features and may change over time.

Vorliq is not a licensed bank, broker, exchange, lender, investment adviser, custodian, or financial institution. Users are responsible for their own keys, local laws, counterparties, posts, offers, votes, and risk decisions. The phrase community savings bank is used only as a metaphor for shared community record keeping.

Anyone can inspect the code, run a node, create a test wallet, explore the chain, try the community features, and contribute feedback. Please read the transparency page, wallet safety guide, and terms before using meaningful wallets.

Discord Announcement
====================

Welcome to Vorliq Version 1.0. Vorliq is experimental open-source community blockchain software with its own native coin, VLQ. The platform includes wallets, mining, sending, community lending-style voting, exchange offers, governance proposals, peer networking, a public node registry, health monitoring, web access, mobile source code, and notifications.
