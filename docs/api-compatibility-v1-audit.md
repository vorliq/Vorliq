# API Compatibility v1 Audit

Vorliq's Node backend exposes about two dozen route groups: chain, wallet, transactions, mining, faucet, lending, exchange, governance, treasury, registry, network peers, audit exports, manifest, profiles, forum, reports, analytics, achievements, incidents, deployment, storage, system, backups, price, and protected admin routes. Flask exposes the underlying blockchain, wallet, transaction, mining, profile, forum, governance, treasury, lending, exchange, registry, faucet, audit, and peer-node routes used by the Node API.

Public read routes include health, deployment, chain summary and blocks, wallet balance, transaction lookup, mining status/history, faucet summary/history, lending/exchange/governance/treasury summaries and public records, registry and network manifests, profiles, forum lists, achievements, analytics summary, storage health, incident status, and public audit exports.

Public write routes include transaction submission, wallet creation, mining attempts, faucet claims, profile saves and wallet-verification submissions, forum posts/replies/votes/tips, reports, analytics events, registry registration/heartbeat, peer announcements, lending votes/repayments, exchange trade actions, governance proposals/votes, treasury proposals/votes, and price signals. Protected admin routes live under `/api/admin` and cover overview, storage, backups, incidents, security, analytics, reports, and moderation.

Before v1 compatibility hardening, many successful responses already included `success`, but error shapes varied between `{ success:false, message }`, `{ success:false, error:"..." }`, upstream Flask payloads, validation responses, and admin-auth responses. Request IDs and explicit API version headers were not present. The most important endpoints to keep stable are `/api/health`, `/api/version`, `/api/chain/summary`, `/api/wallet/balance`, `/api/transaction/send`, `/api/transactions`, `/api/faucet/summary`, `/api/lending/summary`, `/api/exchange/summary`, `/api/governance/summary`, `/api/treasury/summary`, `/api/registry/summary`, `/api/audit/manifest`, `/api/network/manifest`, `/api/profiles`, `/api/forum/posts`, `/api/reports`, and protected `/api/admin/*`.
