Run a Public Vorliq Node

This guide explains how to deploy a public Vorliq node on Ubuntu so other community members can connect to your blockchain API and discover your node through the registry.

What You Need for Production

To run a public Vorliq node, the operator needs a Linux server running Ubuntu 22.04 or later with a public IP address, at least 1 gigabyte of RAM, and at least 10 gigabytes of disk space. A cheap virtual private server from any provider works well for this and usually costs around five dollars per month.

Install Dependencies on the Server

Connect to the Ubuntu server over SSH and run these commands to install Git, nginx, Python 3.12, and Node.js 20. These commands update the server, install system packages, and add the official NodeSource package repository for Node.js 20. 

sudo apt update
sudo apt upgrade -y
sudo apt install -y software-properties-common curl ca-certificates git nginx
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update
sudo apt install -y python3.12 python3.12-venv python3.12-dev
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

Clone and Configure Vorliq

Clone the repository into the server, install the Python, backend, and frontend dependencies, and build the React frontend. For a public node, edit blockchain/app.py and change the Flask host from 127.0.0.1 to 0.0.0.0 so other nodes can reach the blockchain API on port 5001. 

cd /opt
sudo git clone https://github.com/vorliq/Vorliq.git
sudo chown -R $USER:$USER /opt/Vorliq
cd /opt/Vorliq/blockchain
python3.12 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -r requirements.txt
cd /opt/Vorliq/backend
npm install
cd /opt/Vorliq/frontend
npm install
npm run build
nano /opt/Vorliq/blockchain/app.py

Run Vorliq as a Service

Production nodes should run Vorliq through systemd so the Flask API and Node.js backend start automatically when the server boots and restart automatically if they crash. Create the Flask service first, then create the backend service. 

sudo nano /etc/systemd/system/vorliq-blockchain.service
[Unit]
Description=Vorliq Blockchain API
After=network.target

[Service]
WorkingDirectory=/opt/Vorliq/blockchain
ExecStart=/opt/Vorliq/blockchain/.venv/bin/python app.py
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
sudo nano /etc/systemd/system/vorliq-backend.service
[Unit]
Description=Vorliq Backend API
After=network.target vorliq-blockchain.service

[Service]
WorkingDirectory=/opt/Vorliq/backend
Environment=FLASK_URL=http://127.0.0.1:5001
ExecStart=/usr/bin/node index.js
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable vorliq-blockchain vorliq-backend
sudo systemctl start vorliq-blockchain vorliq-backend
sudo systemctl status vorliq-blockchain
sudo systemctl status vorliq-backend

Configure nginx

nginx can serve the React frontend on port 80 and proxy API requests to the Node.js backend. Replace example.com with your domain name if you have one, or use an underscore as the server name when serving directly from an IP address. 

sudo nano /etc/nginx/sites-available/vorliq
server {
    listen 80;
    server_name _;

    root /opt/Vorliq/frontend/build;
    index index.html;

    location / {
        try_files $uri /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:5000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
sudo ln -s /etc/nginx/sites-available/vorliq /etc/nginx/sites-enabled/vorliq
sudo nginx -t
sudo systemctl reload nginx
sudo ufw allow 80/tcp
sudo ufw allow 5001/tcp

Open Your Node to the Network

After the server is running, open the Registry page in the Vorliq app and register your node using your server's public IP address and port 5001, such as http://YOUR_PUBLIC_IP:5001. Then share your node URL in the Vorliq Discord and Telegram so community members can connect to it from their Network page.
